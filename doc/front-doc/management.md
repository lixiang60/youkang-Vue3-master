# 📊 前端项目管理与执行指南

> [!TIP]
> 本文档旨在指导前端团队如何科学、专业地进行项目开发管理，确保开发质量与交付效率。

---

## 1. 进度同步流程 (Progress Sync)

### 1.1 每周一次同步 & 验收
- **汇报核心**：
    1. **本周进展**：完成了哪些模块（对应 `project.md` 状态更新）。
    2. **下周计划**：接下来的里程碑任务重心。
    3. **Bug 回顾**：验收过程发现的问题及修复进度。
    4. **阻塞点 (Blockers)**：目前遇到哪些技术难点或阻碍（如接口，页面实现，逻辑梳理等）。

### 1.2 视觉化管理
- **动态更新**：按天更新 `project.md` 中的“接口状态”列，确保项目所有参与者信息对称。

---

## 2. 协作与沟通规范 (Collaboration)

### 2.1 跨团队协作 (前端 x 后端 x 验收)
- **高效沟通**：复杂逻辑沟通直接预约 **腾讯会议**，@所有人。会后输出简要的 **会议纪要** 或写入在线文档。
- **文档联动机制**：
    - **BUG 处理**：李姗姗验收发现的问题 -> 记录至 BUG 在线文档并 @程荣禄 -> 程分析原因转 @李翔 -> 李确认后处理。
    - **接口联调**：联调问题优先在线记录并 @相关人（非实时沟通）；同时在线时可直接语音/文字对齐。
    - **接口变动**：接口字段或路径任何变动，**必须更新技术文档**。

---

## 3. 项目计划里程碑 (Milestones)

根据 `project.md` 模块拆解，项目分为四个关键阶段及其分代版本：

### 3.1 T1：客户设置
- **交付范围**：
    - 4.1 客户管理：添加/编辑、密码修改、提醒功能。
    - 4.2 课题组管理：费用导入/导出、价格设置、提醒设置。
    - 4.4 客户审核：审核转化、用户暂存机制。

### 3.2 T2：测序管理 (标准)
- **交付范围**：
    - 5.1 - 5.2：订单录入、样品管理、实验室调拨。
    - 5.3 - 5.7：模板排版、反应生产、测序 BDT 表生成。
    - 5.10 - 5.12：报告生产、文件上传、报告邮件自动下发。

### 3.3 T2：测序管理 (四代)
- **交付范围**：
    - 基于四代测序特有逻辑的订单录入与样品流转。
    - 四代产出的特定数据报告生成与上传。
    - 差异化邮件通知模板。

### 3.4 T3：合成管理 (标准)
- **交付范围**：
    - 6.1 - 6.5：合成订单管理、样品编辑、安排合成计划。
    - 6.9 - 6.11：合成分装值导入、机器分装、烘干出库流程。
    - 6.12 - 6.13：合成订单出库、送货单打印。

### 3.5 T3：合成管理 (四代)
- **交付范围**：
    - 四代合成订单的修饰、氨解及纯化状态跟踪。
    - 匹配四代工艺的分装值导入与自动化分装。
    - 适配四代订单的确认单与结算逻辑。

### 3.6 T4：订单结算与财务
- **交付范围**：
    - 5.15 & 6.14：订单款项结算、财务审核返还、开票管理。
    - 全系统 Bug 修复：根据线上 BUG 文档完成所有 P0/P1 问题闭环。
