# knowledge-precipitator

## 目标
自动化的知识沉淀与规则优化专家。通过深度分析当前的开发活动（包括聊天上下文、代码 Diff、技术方案等），将散落的决策点沉淀为结构化的项目规则，确保后续开发的强一致性。

## 使用场景
- 当完成了一个复杂模块的开发，需要总结其中的设计模式。
- 当发现代码中存在反复纠正的风格问题，需要写入 `rules.md`。
- 当用户手动触发 `/knowledge-sync` 时，对当前进度进行全局知识同步。

## 核心能力
1. **上下文溯源**：扫描最新的 `overview.txt` 聊天日志，提取技术决策。
2. **变更分析**：通过 `git diff` 或文件快照对比，识别代码中的“黄金模式”。
3. **规则迭代**：更新或新建 `.agents/rules/` 下的规范文档。
4. **安全确认**：每次修改规则文件前，均会展示变更 Diff 并请求用户确认。

## 工作流程

### 1. 扫描与提取
- 读取最近的聊天记录片段。
- 检查当前未提交或最近提交的代码变更。
- 检索工作区中的 `/docs` 或 README。

### 2. 提炼知识项
- 识别新引入的技术栈或库组件。
- 提取特定的 UI 布局逻辑（如表格间距、表单栅格）。
- 识别 API 交互模式。

### 3. 规则优化建议
- 根据提炼的内容，生成对应的 Markdown 规则片段。
- 匹配已有的规则文件（如 `table-style.md`, `dev-rules.md`）。

### 4. 交互式更新
- 展示建议的变更。
- **提示用户**：“检测到新的开发模式，是否将其合并入项目规范中？”。
- 用户确认后，使用 `multi_replace_file_content` 执行更新。

## 注意事项
- 严禁删除用户手动录入的历史关键规则。
- 优先保留 `Persistent Context` 中的知识原子。
- 保持规范文档的紧凑性，单文件建议不超过 2000 词。
