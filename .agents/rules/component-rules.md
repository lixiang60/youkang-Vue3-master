# 🧩 组件与页面编码规范 (Component & Coding Guidelines)

本规范定义了 Vue 组件开发中的结构、交互及按钮样式标准，以确保全站视觉与逻辑的一致性。

## 1. 📂 页面视图与代码结构规范 (Page Anatomy)

为了应对超长文件（如含复杂业务流的页面）的阅读疲劳与维护困难，不论是在 `<template>` 还是 `<script setup>` 中，都**必须严格进行物理分块**注释。

### 1.1 视图层区块 (`<template>`)
在模板部分，要求严格按照页面的物理视觉从上到下进行注释分隔：
```html
<!-- 1. 查询面板 -->
<!-- 2. 操作按钮区 -->
<!-- 3. 数据表格区 -->
<!-- 4. 基础CRUD对话框 -->
<!-- 5. 业务操作对话框 (如排版忽略、添加板号等特定业务弹窗独立分块) -->
<!-- 6. 其他公共组件/外部组件 -->
```

### 1.2 逻辑层区块 (`<script setup>`)
在引入文件之后，代码必须使用 `// --- N. 模块名 ---` 的注释形式，将逻辑死死切割为以下 6 大功能块：

```javascript
import { ... } from 'vue'

// --- 1. Constants & Config (静态常量与配置) ---
// 包括：路由信息提取、当前代理引用 proxy 提取、固定字典/常量。
// 必须放置庞大且结构固定的主要配置：columns(表头), searchFields(搜索项), rules(校验规则)。

// --- 2. State (响应式状态) ---
// 包括核心状态(loading, total)、勾选状态(ids, single, multiple)。
// 集中管理所有的弹窗显隐控制位(openXXX)与各类表单绑定模型参数(form, queryParams 等)。

// --- 3. Computed (派生属性) ---
// 包括所有依赖源数据动态计算派生得出的状态。

// --- 4. Methods (方法逻辑) ---
// 核心逻辑集中区，内部必须采用 /** 业务类型 */ 进一步划分子块，如：
// /** 初始化与查询 */ 
// /** 表格与选框交互 */ 
// /** 基础 CRUD */ 
// /** 业务操作 - 添加板号 */ (独立业务闭环)

// --- 5. Lifecycle (生命周期) ---
// 统一将 onMounted, onActivated 等钩子沉底放置。

// --- 6. Watchers (侦听器) ---
// (可选) 放置深度监听逻辑，例如对 columns 监听以进行本地 localStorage 同步的任务。
```

## 2. 💬 对话框 (Dialog) 规范

- **宽度与层级**：统一使用 `append-to-body`，避免层级遮挡。宽度根据复杂度建议使用 `600px` - `1000px` 或百分比。
- **按钮规范**：
    - **确定按钮**：统一使用 `type="success" :icon="Check"`，标签为 `确定`（无空格）。
    - **取消按钮**：统一使用 `type="danger" :icon="Close"`，标签为 `取消`（无空格）。
- **底部容器**：必须使用 `<div class="dialog-footer" style="text-align: center">` 包裹操作按钮。

## 3. 🔘 按钮与图标规范

- **图标使用**：所有图标必须使用 `element-plus/icons-vue` 库。
- **配置方式**：推荐采用组件/变量导入方式（如 `:icon="Search"`），而非字符串形式，以保证类型安全和样式规范。
- **按钮尺寸**：在表格工具栏及弹窗页脚中，统一使用 `size="small"` 以维持紧凑布局。

## 4. 📝 表单与校验规范

- **Scoped 约束**：由于是公共/后台系统，组件内部自定义样式必须使用 `<style scoped>`，严禁污染全局。
- **表单布局首选**：弹窗内部的表单推荐优先使用 Element 原生 `el-form-item` 搭配 `el-row :gutter="20"`，留出充足的呼吸感（禁用 `size="small"` 等紧凑属性）。
- **慎用强排版**：带边框的紧凑型 `.well-form` 网格布局应**仅限于**打印参数设定的极端高密度场景，绝大多数 CRUD 弹窗不应使用。

## 5. ⚠️ 业务确认与反馈规范

- **核心动作确认**：涉及删除、结算、批量修改等具有破坏性或流程转折性的操作，必须使用 `proxy.$modal.confirm('确认消息')` 弹出提示，严禁静默执行。
- **状态反馈**：异步操作执行中需挂载 `v-loading`；执行完成后需调用 `proxy.$modal.msgSuccess()` 给出明确反馈。

## 6. 🖨 报表与打印规范

- **字体约束**：正式报表、送货单、报告单必须强制使用 `font-family: 'SimSun', serif;`（宋体），确保打印预览与纸质输出的一致性。
- **布局抽离**：复杂的打印 HTML 片段必须抽离为业务对应的独立组件，通过 `v-print` 指令或 `PrintDialog` 触发。
- **视觉一致性**：打印容器样式必须处理 `@media print` 下的分页符 (`page-break-after: always`) 并隐藏所有非打印 UI 元素（如按钮、背景区）。
